# Instructions for Agents Working on the Docs Website

This directory contains Chatto's public documentation website, built with Astro and Starlight.

## Audience

Write for Chatto users, server operators, administrators, and API consumers. Do not write visible page content for Chatto repository maintainers or contributors.

Avoid public-facing instructions such as "edit this file", "run codegen", "do not edit generated output", or other internal workflow notes. Hidden source comments for maintainers are fine when useful.

## Style

- Be direct and concise.
- Use second person, present tense.
- Lead with what the reader needs to know.
- Use tables and short lists instead of long prose.
- Keep generated API reference pages useful for client authors, not for Chatto developers maintaining the protobuf files.

## Navigation

The Starlight sidebar is configured in `astro.config.mjs`. Add new pages there unless the section intentionally uses autogenerated entries.
