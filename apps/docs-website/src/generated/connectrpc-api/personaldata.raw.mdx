## PersonalDataService

PersonalDataService stores the authenticated user's portable preferences and
server directory on their chosen home server.

<a id="chatto-personaldata-api-v1-PersonalDataService-GetPreferences"></a>

### GetPreferences

Gets portable preferences for the authenticated user.

```http
POST /api/connect/chatto.personaldata.api.v1.PersonalDataService/GetPreferences
```

<a id="chatto-personaldata-api-v1-GetPreferencesRequest"></a>

#### Input: GetPreferencesRequest

GetPreferencesRequest selects the authenticated user's preferences.


<a id="chatto-personaldata-api-v1-GetPreferencesResponse"></a>

#### Result: GetPreferencesResponse

GetPreferencesResponse returns the authenticated user's portable preferences.

| Field | Type | Description |
| --- | --- | --- |
| `preferences` | [`Preferences`](#chatto-personaldata-api-v1-Preferences) | Current preferences; present even when no fields have been set. |

## Supporting Types

<a id="chatto-personaldata-api-v1-KnownServer"></a>

### KnownServer

KnownServer describes one server in the user's portable server directory.
Authentication credentials are device-local and are never included.

| Field | Type | Description |
| --- | --- | --- |
| `id` | `string` | Opaque identifier within this portable directory. Clients must match servers by canonical URL origin rather than treating this as global identity. |
| `url` | `string` | Canonical HTTP or HTTPS origin for the server. |
| `name` | `string` | Last discovered display name. |
| `icon_url` | `optional string` | Last discovered server icon URL. |
| `added_at` | `google.protobuf.Timestamp` | Time this entry was first added to the portable directory. |



<a id="chatto-personaldata-api-v1-Preferences"></a>

### Preferences

Preferences contains personal settings which should follow the authenticated
user between Chatto clients.

| Field | Type | Description |
| --- | --- | --- |
| `locale` | `optional string` | BCP 47 locale tag, such as en-GB or de. |
| `timezone` | `optional string` | IANA timezone name. Empty means to use the client timezone. |
| `time_format` | optional [`TimeFormat`](#chatto-personaldata-api-v1-TimeFormat) | Preferred time display format. |
