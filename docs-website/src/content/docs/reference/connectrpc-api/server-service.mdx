## Services

<a id="chatto-api-v1-ServerService"></a>

### ServerService

Provides public server discovery metadata.

<a id="chatto-api-v1-ServerService-GetServer"></a>

#### GetServer

Returns public server metadata, branding, registration status, and login
provider information. This RPC is available before login so clients can
render the first server screen and authentication choices.

```http
POST /api/connect/chatto.api.v1.ServerService/GetServer
```

<a id="chatto-api-v1-GetServerRequest"></a>

##### Input: GetServerRequest

Request for public server metadata.


<a id="chatto-api-v1-GetServerResponse"></a>

##### Result: GetServerResponse

Public metadata clients need before they can authenticate or render the
initial server view.

| Field | Type | Description |
| --- | --- | --- |
| `name` | `string` | Display name of the Chatto server. |
| `version` | `string` | Server software version. |
| `auth_methods` | `repeated string` | Enabled legacy authentication method identifiers. |
| `auth_providers` | `repeated AuthProvider` | Configured login providers. |
| `registration_open` | `bool` | Whether users can create accounts through the public UI. |
| `authorize_url` | `string` | URL for the legacy authorization flow, when enabled. |
| `welcome_message` | `string` | Optional welcome message configured by the server administrator. |
| `description` | `string` | Optional server description configured by the server administrator. |
| `icon_url` | `string` | Optional server icon URL suitable for branding the server in client UI. |
| `banner_url` | `string` | Optional server banner URL suitable for larger server branding surfaces. |



## Supporting Types

<a id="chatto-api-v1-AuthProvider"></a>

#### AuthProvider

Describes one login provider exposed by the server discovery endpoint.

Clients can render these providers before a user has authenticated.

| Field | Type | Description |
| --- | --- | --- |
| `id` | `string` | Stable provider identifier used by clients when starting login. |
| `type` | `string` | Provider category, such as "password" or "oidc". Clients should treat unknown provider types as unsupported. |
| `label` | `string` | Human-readable provider label for login UI. |
| `login_url` | `string` | URL that starts login for this provider. |
