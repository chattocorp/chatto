## MessageService

Creates messages in room and thread timelines.

<a id="chatto-api-v1-MessageService-CreateMessage"></a>

### CreateMessage

Creates a message for the current user. The user must be a room member and
must have message.post for room messages or message.post-in-thread for
thread replies. Echoing a thread reply also requires message.echo and
message.post.

```http
POST /api/connect/chatto.api.v1.MessageService/CreateMessage
```

<a id="chatto-api-v1-CreateMessageRequest"></a>

#### Input: CreateMessageRequest

Request to create a message in a room or thread.

| Field | Type | Description |
| --- | --- | --- |
| `room_id` | `string` | Required. Room where the message should be created. |
| `body` | `string` | Message body text. Required unless attachments or attachment_asset_ids is non-empty. |
| `attachment_asset_ids` | `repeated string` | Existing room-scoped attachment asset IDs to include with the message. |
| `thread_root_event_id` | `string` | Event ID of the thread root message when posting a thread reply. |
| `in_reply_to` | `string` | Event ID this message replies to for attribution. |
| `also_send_to_channel` | `bool` | True to also echo a thread reply into the main room timeline. |
| `mention_confirmation_token` | `string` | Short-lived token returned by a prior mention confirmation challenge. |
| `link_preview` | [`LinkPreview`](/reference/connectrpc-api/types/#chatto-api-v1-LinkPreview) | Link preview selected by the client. Server response-only LinkPreview fields are ignored. |
| `attachments` | repeated [`MessageAttachmentUpload`](/reference/connectrpc-api/types/#chatto-api-v1-MessageAttachmentUpload) | Browser-uploaded attachments to include with the message. |


<a id="chatto-api-v1-CreateMessageResponse"></a>

#### Result: CreateMessageResponse

Result of creating a message.

| Field | Type | Description |
| --- | --- | --- |
| `result.event` | [`RoomTimelineEvent`](/reference/connectrpc-api/types/#chatto-api-v1-RoomTimelineEvent) | Renderable timeline event for the created message. |
| `result.mention_confirmation` | [`MentionConfirmationChallenge`](/reference/connectrpc-api/types/#chatto-api-v1-MentionConfirmationChallenge) | Challenge shown when a message would notify many people. |
| `includes` | [`RoomTimelineIncludes`](/reference/connectrpc-api/types/#chatto-api-v1-RoomTimelineIncludes) | Related entities needed to render event when a message was created. |

### UpdateMessage

Edits a message body. Authors can edit their own messages within the edit
window. Non-authors need message.manage and cannot change channel echo
state.

```http
POST /api/connect/chatto.api.v1.MessageService/UpdateMessage
```

<a id="chatto-api-v1-MessageService-RefreshMessageAttachmentUrls"></a>

### RefreshMessageAttachmentUrls

Refreshes signed URLs for the current attachments on one message.
Authentication and room membership are required. Returns NOT_FOUND when the
message does not exist, is not a current visible message, or belongs to a
different room. Returns an empty attachment list when the message exists
but currently has no attachments.

```http
POST /api/connect/chatto.api.v1.MessageService/RefreshMessageAttachmentUrls
```

<a id="chatto-api-v1-RefreshMessageAttachmentUrlsRequest"></a>

#### Input: RefreshMessageAttachmentUrlsRequest

Request to refresh signed attachment URLs for one message.

| Field | Type | Description |
| --- | --- | --- |
| `room_id` | `string` | Required room ID. |
| `event_id` | `string` | Required message event ID. |
| `thumbnail` | [`AttachmentThumbnailOptions`](/reference/connectrpc-api/types/#chatto-api-v1-AttachmentThumbnailOptions) | Thumbnail URL options. Defaults are applied when absent. |


<a id="chatto-api-v1-RefreshMessageAttachmentUrlsResponse"></a>

#### Result: RefreshMessageAttachmentUrlsResponse

Response containing fresh signed URLs for the current message attachments.

| Field | Type | Description |
| --- | --- | --- |
| `attachments` | repeated [`RefreshedAttachmentUrls`](/reference/connectrpc-api/types/#chatto-api-v1-RefreshedAttachmentUrls) | Fresh URL bundle for each current attachment on the message. |


<a id="chatto-api-v1-MessageService-BatchRefreshMessageAttachmentUrls"></a>

### BatchRefreshMessageAttachmentUrls

Refreshes signed URLs for the current attachments on multiple messages in
one room. Authentication and room membership are required. Missing,
retracted, non-message, and wrong-room event IDs are omitted. Results
preserve first-seen request order and repeated event IDs are de-duplicated.
Existing visible messages with no current attachments are returned with an
empty attachment list.

```http
POST /api/connect/chatto.api.v1.MessageService/BatchRefreshMessageAttachmentUrls
```

<a id="chatto-api-v1-BatchRefreshMessageAttachmentUrlsRequest"></a>

#### Input: BatchRefreshMessageAttachmentUrlsRequest

Request to refresh signed attachment URLs for many messages in one room.

| Field | Type | Description |
| --- | --- | --- |
| `room_id` | `string` | Required room ID. |
| `event_ids` | `repeated string` | Required message event IDs. Missing, retracted, non-message, and wrong-room event IDs are omitted from the response. |
| `thumbnail` | [`AttachmentThumbnailOptions`](/reference/connectrpc-api/types/#chatto-api-v1-AttachmentThumbnailOptions) | Thumbnail URL options. Defaults are applied when absent. |


<a id="chatto-api-v1-BatchRefreshMessageAttachmentUrlsResponse"></a>

#### Result: BatchRefreshMessageAttachmentUrlsResponse

Response containing fresh signed URLs for many visible messages.

| Field | Type | Description |
| --- | --- | --- |
| `messages` | repeated [`RefreshedMessageAttachmentUrls`](/reference/connectrpc-api/types/#chatto-api-v1-RefreshedMessageAttachmentUrls) | Refreshed messages in first-seen request order. |
