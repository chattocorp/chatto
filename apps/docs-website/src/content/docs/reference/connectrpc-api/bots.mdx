## BotService

Manages bot accounts owned by the authenticated human or, with
administrative authority, by other users.

<a id="chatto-api-v1-BotService-ListBots"></a>

### ListBots

Lists bots the caller may manage. Owners need `bot.create`; administrators
need `bot.manage` to see bots owned by other users.

```http
POST /api/connect/chatto.api.v1.BotService/ListBots
```

<a id="chatto-api-v1-ListBotsRequest"></a>

#### Input: ListBotsRequest

Request the bot accounts manageable by the authenticated user.

| Field | Type | Description |
| --- | --- | --- |
| `search` | `string` | Optional case-insensitive search against login and display name. |
| `page` | [`PageRequest`](/reference/connectrpc-api/types/#chatto-api-v1-PageRequest) | Page request. Defaults to 20 results when absent or limit is zero. |


<a id="chatto-api-v1-ListBotsResponse"></a>

#### Result: ListBotsResponse

Page of manageable bot accounts.

| Field | Type | Description |
| --- | --- | --- |
| `bots` | repeated [`Bot`](/reference/connectrpc-api/types/#chatto-api-v1-Bot) | Bots in the requested page. |
| `page` | [`PageInfo`](/reference/connectrpc-api/types/#chatto-api-v1-PageInfo) | Page metadata. |


<a id="chatto-api-v1-BotService-GetBot"></a>

### GetBot

Gets one manageable bot. Returns NOT_FOUND for an unknown account and
PERMISSION_DENIED when the bot exists but the caller may not manage it.

```http
POST /api/connect/chatto.api.v1.BotService/GetBot
```

<a id="chatto-api-v1-GetBotRequest"></a>

<a id="chatto-api-v1-BotService-BatchGetBots"></a>

### BatchGetBots

Gets manageable bots for multiple IDs. Unknown and inaccessible bots are
omitted so one missing record does not fail the batch.

```http
POST /api/connect/chatto.api.v1.BotService/BatchGetBots
```

<a id="chatto-api-v1-BatchGetBotsRequest"></a>

#### Input: BatchGetBotsRequest

Request bot records for a set of stable account IDs.

| Field | Type | Description |
| --- | --- | --- |
| `bot_ids` | `repeated string` | Required bot account IDs. Unknown or inaccessible IDs are omitted. |

<a id="chatto-api-v1-BotService-RotateBotAPIKey"></a>

### RotateBotAPIKey

Issues the bot's first API key or replaces its existing key. The previous
key is invalidated immediately and the new secret is returned only once.

```http
POST /api/connect/chatto.api.v1.BotService/RotateBotAPIKey
```

<a id="chatto-api-v1-RotateBotAPIKeyRequest"></a>

#### Input: RotateBotAPIKeyRequest

Request replacement of one manageable bot's API key.

| Field | Type | Description |
| --- | --- | --- |
| `bot_id` | `string` | Required bot account ID. |


<a id="chatto-api-v1-RotateBotAPIKeyResponse"></a>

#### Result: RotateBotAPIKeyResponse

Show-once bot API-key response.

| Field | Type | Description |
| --- | --- | --- |
| `bot` | [`Bot`](/reference/connectrpc-api/types/#chatto-api-v1-Bot) | Bot with current non-secret API-key metadata. |
| `api_key` | `string` | Newly issued raw API key. Chatto does not store or return it again. |
