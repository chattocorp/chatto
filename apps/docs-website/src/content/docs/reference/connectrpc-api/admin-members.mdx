## AdminMemberService

Server-admin user identity and cooldown commands.

On the public web listener, these RPCs use normal user authentication and
RBAC checks. On the dedicated Admin API listener, configured operator tokens
may call the same RPCs from allowed CIDRs as Chatto's system actor.

<a id="chatto-admin-v1-AdminMemberService-CreateUser"></a>

### CreateUser

Creates a user as an authenticated admin or local operator.

```http
POST /api/connect/chatto.admin.v1.AdminMemberService/CreateUser
```

<a id="chatto-admin-v1-CreateUserRequest"></a>

#### Input: CreateUserRequest

Request to create a user as an admin or operator action.

| Field | Type | Description |
| --- | --- | --- |
| `login` | `string` | Required login for the new user. |
| `display_name` | `string` | Optional display name. Defaults to login when empty. |
| `password` | `string` | Optional password. When present, it must satisfy server password policy. |
| `verified_email` | `string` | Optional email to add as already verified. |
| `role_names` | `repeated string` | Optional explicit server roles to assign after creation. |
